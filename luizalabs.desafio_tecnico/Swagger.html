A autenticação de todos os endpoints segue o padrão JWT, para gerar um token use o endpoint Login, esse endpoint exige que seja colocado o usuário e senha no formato basic codificado no padrão base64 via cabeçalho de autorização, como no exemplo a seguir


header: authorization: basic BASE64({user_name};{password})


user_name: dani@travatech.com.br

password: 3\)?q)#A?sKw+N?(


concatenar: dani@travatech.com.br;3\)?q)#A?sKw+N?(

base64: ZGFuaUB0cmF2YXRlY2guY29tLmJyOzNcKT9xKSNBP3NLdytOPyg=


authorization: basic ZGFuaUB0cmF2YXRlY2guY29tLmJyOzNcKT9xKSNBP3NLdytOPyg=


Para esse desafio não foi considerado a autenticação do usuário, então qualquer user_name e password informado vão ser autenticados.


Para fazer o upload do arquivo use o endpoint Legacy passando o arquivo no corpo com o parâmetro “file”, todo o processamento é assíncrono, então para acompanhar a requisição use o request_id passado na resposta ne endpoint de get, após o processamento concluído (ou durante, porem com dados parciais) os dados podem ser acessados pelo endpoint de User/Legacy, ele ira mostrar no forma do legado, os demais endpoints será exibido a nova estrutura
